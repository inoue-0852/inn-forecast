# 月別宿泊数予測デモ — 説明書

**対象ファイル**: `20260225_demo_script.ipynb`
**目的**: 島根県・出雲市・大社町の観光動態データと旅館の宿泊実績を組み合わせ、2026年の月別宿泊人数を予測する

---

## 概要

大社町のある旅館の「過去2年間（2024〜2025年）の宿泊実績」と、周辺地域の公開観光統計データを突き合わせ、**季節変動パターンをベースにした2026年の月別宿泊予測**を行うデモスクリプトです。

PoCフェーズ（現状把握・可視化）からMVPフェーズ（予測モデルの実装）までを1本のNotebookで通しています。

### セル 8〜9 — データクレンジング

バラバラな形式のデータを分析しやすい形式に統一します。実務において最も工数がかかる工程です。

**主な変換処理**

```
和暦（R5, R6）
    ↓  convert_era_to_year()
西暦（2023, 2024）

横持ち（1月|2月|...|12月 が列）
    ↓  tidy_up_monthly_columns() ＝ pandas melt()
縦持ち（month列にYYYY-MM形式）

複数テーブル
    ↓  pandas merge（LEFT JOIN / month キー）
統合テーブル（旅館実績 + 3地域の観光データ）
```

**統合後のテーブル構造**

| 列名 | 内容 |
|---|---|
| `month` | 年月（YYYY-MM） |
| `ryokan_stays` | 旅館の月別宿泊人数 |
| `shimane_visitors` | 島根県 月別観光客数（合計） |
| `izumo_visitors` | 出雲市 月別観光客数（合計） |
| `taisha_stays` | 大社町 月別宿泊者数（合計） |
| `year_only` | 年（文字列） |
| `month_only` | 月（文字列 01〜12） |

> 欠損値は `ffill`（前月値で補完）→ `fillna(0)` の順で処理されます。
> データ期間：2024-01 〜 2025-12（24ヶ月）

### セル 10〜11 — データ可視化（PoCフェーズ）

過去データの傾向と季節性を2つのグラフで確認します。

**グラフ1：観光動態と旅館宿泊数の連動確認**

- 左Y軸：旅館の宿泊数（赤実線）
- 右Y軸：出雲市・島根県の観光客数（青・緑の点線）
- 目的：地域の観光トレンドと旅館宿泊数の相関を目視確認する

**グラフ2：年別の月別宿泊数比較（季節性の把握）**

- X軸：月（01〜12）
- Y軸：宿泊人数
- 2024年・2025年の折れ線を重ねて表示
- 目的：「何月に宿泊が増えるか」という季節パターンの確認

---

### セル 12〜13 — 予測・可視化（MVPフェーズ）

過去2年の月別平均を「季節係数」として使い、2026年の12ヶ月分を予測します。

**予測ロジック**

```
① 月ごとの平均を算出
   average_seasonal_stays = df.groupby('month_only')['ryokan_stays'].mean()
   例：1月平均 = (2024年1月実績 + 2025年1月実績) ÷ 2

② 上限キャップを適用
   predicted = average_seasonal_stays.clip(upper=1450)

③ 予測結果を実績グラフに接続して表示
```

**グラフ：実績 + 予測の時系列**

- 青実線：2024〜2025年の実績
- オレンジ点線：2026年の予測値
- 赤点線：施設上限（1,450人/月）
- オレンジ帯：予測の幅（±10%）

MVP（最小実用版）フェーズ
  → セル6：季節平均という最もシンプルな手法で予測を出す
